# 示例棋牌项目 Agent 知识库

## 1. 目录说明

本目录用于沉淀 Agent 可检索、可回答、可复用的标准知识条目。知识条目来自游戏规则截图 OCR、人工整理规则原文和后续产品同学补充说明。

当前文件：

| 文件 | 内容 | 状态 |
| --- | --- | --- |
| [CARD_GAME_KB_SEED.md](CARD_GAME_KB_SEED.md) | 第一批高价值知识库条目 | 待人工抽查后进入 MVP 检索库 |
| [KB_ACCEPTANCE_TESTS.md](KB_ACCEPTANCE_TESTS.md) | 知识库验收问题与转反馈样例 | 可用于 Agent 回答测试 |

配套机器可读文件：

| 文件 | 内容 | 状态 |
| --- | --- | --- |
| `data/knowledge_base/card_game_kb_seed.json` | 第一批知识库 JSON 数据 | 可作为后续 mock 检索库输入 |

## 2. 条目状态

| 状态 | 含义 | Agent 使用策略 |
| --- | --- | --- |
| 完整 | 截图规则链路较完整，可用于回答 | 可直接检索回答，但仍需提示以游戏内页面为准 |
| 部分缺失 | 缺少时间、版本、奖励配置或边界条件 | 回答时加入不确定边界，必要时追问 |
| 待确认 | OCR 表格、图片牌型、遮挡文字等存在风险 | 不用于强确定性回答，只作为线索 |

## 3. 入库原则

- 优先收录玩家高频问题：玩法规则、奖励是否清空、活动奖励未到账、比赛规则。
- 每条知识都必须包含不可回答边界，避免 Agent 对补偿、单局结算、账号处罚等问题直接下结论。
- 对涉及活动配置、概率、奖励、价格的回答，统一提示“以游戏内当前活动页面为准”。
- 对涉及单局争议、到账争议、奖励争议的问题，引导玩家提交反馈并补充截图、时间、牌局 ID 或订单信息。
