# PRD：游戏智能客服与玩家反馈分析 Agent

## 1. 背景

示例棋牌项目类棋牌游戏的客服入口承载了规则咨询、活动答疑、充值问题、BUG 反馈、举报投诉和体验建议等复杂需求。传统客服系统通常只解决“玩家问答”或“人工工单”，但产品、运营、策划和技术团队更需要从玩家反馈中发现高优问题、高频诉求和可转化的优化方向。

本项目希望在短周期内完成一个可运行、可复用的 AI Agent MVP，用“玩家客服聊天页 + 结构化反馈 + 协同系统预留 + 知识库更新 SOP”串起从玩家问题到团队行动的闭环。

## 2. 一句话描述

面向示例棋牌项目玩家提供规则咨询、活动答疑、充值/BUG/举报/建议反馈收集，并将玩家反馈结构化同步给客服、策划、运营和技术同学，帮助团队识别高优问题、高频诉求和产品优化方向。

## 3. 产品目标

### 3.1 玩家目标

- 快速获得玩法、活动、奖励等问题的明确答复。
- 对充值、奖励、BUG、举报、账号等问题能按要求提交必要信息。
- 提交后拿到反馈编号，并能查询处理进度。

### 3.2 团队目标

- 自动获得结构化反馈字段，减少客服二次整理成本。
- 对 P0/P1 问题及时告警，避免支付、登录、崩溃和舆情问题扩散。
- 每日沉淀反馈趋势、高频诉求和 AI 产品建议。
- 将重复咨询和规则误解反哺知识库。

### 3.3 复用目标

- 沉淀场景定义、流程设计、字段建模、规则引擎和协同工作流的通用方法。
- 形成可复用模板，未来可快速迁移到其他游戏项目。

## 4. 用户角色

| 角色 | 核心诉求 | Agent 价值 |
| --- | --- | --- |
| 玩家 | 快速咨询、提交问题、查询进度 | 即时问答、自动追问、生成反馈 ID |
| 客服 | 处理 AI 无法解决的问题 | 获得完整字段、摘要、优先级和处理建议 |
| 游戏策划 | 发现规则误解和玩法建议 | 高频问题聚类、规则库补充建议 |
| 运营 | 关注活动、奖励、情绪和舆情 | P0/P1 告警、日报和活动问题摘要 |
| 技术/测试 | 定位 BUG、闪退、卡死、支付异常 | 收集版本、设备、场景、牌局 ID、截图等排查信息 |

## 5. MVP 支持场景

| 场景 | 玩家示例 | Agent 处理方式 |
| --- | --- | --- |
| 玩法规则 | 黄金牌活动炸弹怎么算？ | 检索规则库并回答 |
| 活动规则 | 连续挑战活动鱼票会清空吗？ | 检索活动规则，提示以游戏内页面为准 |
| 充值支付 | 我充了 30 元没到账 | 收集金额、时间、渠道、订单信息，创建反馈 |
| 活动奖励 | 活动奖励没发 | 收集活动名称、完成时间、奖励内容，创建反馈 |
| 游戏 BUG | 对局卡死了 | 收集版本、设备、场景、截图/牌局 ID |
| 账号安全 | 账号登不上去了 | 引导验证，创建人工反馈 |
| 举报投诉 | 有人开挂/骂人 | 收集被举报用户、牌局 ID、时间、截图 |
| 体验建议 | 匹配太慢了 | 记录建议，按频次和影响面判断优先级 |

## 6. MVP 不做什么

- 不接真实订单查询，只做字段收集和人工处理入口。
- 不做完整客服后台，只做反馈结构化、状态流转和同步预留。
- 不做复杂权限系统，默认内部协作由飞书承接。
- 不做大型数据看板，优先输出日报和简表。
- 不做多项目后台，先通过配置模板保留可扩展性。

## 7. 玩家端流程

```text
玩家进入客服入口
-> Agent 展示欢迎语和快捷问题
-> 玩家输入问题或选择快捷入口
-> Agent 识别意图、实体、情绪和风险
-> 若为规则/活动咨询，检索知识库并回答
-> 若信息不足，继续追问关键字段
-> 若涉及风险或未解决，生成反馈确认卡
-> 玩家确认提交
-> 系统生成反馈 ID
-> 同步飞书反馈表
-> 若 P0/P1，触发飞书群通知
-> 玩家可查询处理进度
```

## 8. 策划运营侧流程

```text
反馈入库
-> Agent 自动生成摘要、分类、标签、优先级
-> 同类问题聚合与趋势判断
-> P0 直接飞书群 @所有人
-> P1 @对应负责人或值班同学
-> P2/P3 进入反馈文档和日报
-> 每日生成简报
-> 策划/运营/客服/技术处理
-> 处理结论回写
-> 高频问题沉淀为需求或知识库更新建议
```

## 9. 核心功能需求

### 9.1 聊天入口

- 展示欢迎语和快捷入口。
- 支持玩家自由输入。
- 支持反馈确认卡。
- 支持反馈提交成功后的 ID 展示。
- 支持按反馈 ID 查询处理状态。

### 9.2 意图识别

需要识别：

- 咨询类：玩法规则、活动规则。
- 反馈类：充值支付、活动奖励、BUG、账号安全、举报投诉、体验建议。
- 风险类：投诉、退款、未成年人支付、舆情扩散、无法登录、大面积异常。

### 9.3 知识库问答

- 从标准知识条目中检索相关规则。
- 输出简洁、明确、面向玩家的回答。
- 对活动规则类问题提示“以游戏内活动页面为准”。
- 当资料缺失或不确定时，说明无法确认并引导提交反馈或联系人工客服。

### 9.4 反馈结构化

- 自动生成反馈 ID。
- 记录玩家基础信息、问题理解字段和协同字段。
- 自动生成 AI 摘要、分类、子类、标签、优先级和置信度。
- 信息不足时按场景追问关键字段。

### 9.5 优先级判断

- 根据分类、关键词、情绪、玩家分层、版本集中度和同类反馈量判断 P0-P3。
- P0 立即告警。
- P1 高优处理。
- P2 进入常规跟进。
- P3 记录观察和知识库优化。

### 9.6 飞书协同

- P0 群聊 @所有人，发送告警卡片。
- P1 @对应负责人或值班同学。
- P2/P3 写入飞书文档或多维表格。
- 每日生成玩家反馈简报。
- 告警需按规则去重和聚合。

### 9.7 知识库更新

- 支持产品同学提供文字或截图。
- AI 提取规则信息并生成标准知识条目。
- 补充常见问法、标准回答、澄清策略和不可回答边界。
- 人工确认后同步规则文档和 Agent 检索库。

## 10. 成功指标

MVP 阶段指标：

- 能覆盖 8 类核心问题场景。
- 10-20 个典型玩家问题可被正确分类。
- 规则类问题能返回知识库答案。
- 支付、账号、BUG、举报等问题能触发追问和反馈卡。
- P0/P1 可以按规则生成告警内容。
- 每日简报能基于 mock 反馈生成结构化输出。

产品阶段可观测指标：

- AI 自动解决率。
- 转人工率。
- P0/P1 响应时长。
- 反馈字段完整率。
- 高频问题知识库命中率。
- 玩家负面情绪占比。

## 11. 风险与边界

- 订单、账号、处罚等高敏问题不直接给最终结论，只负责收集信息并转人工。
- 活动规则存在版本和配置差异时，以游戏内页面和官方公告为准。
- Agent 不承诺赔付、退款、解封等处理结果。
- 涉及隐私、账号安全、未成年人支付等问题必须进入人工审核。
