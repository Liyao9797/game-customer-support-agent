# 优先级判断规则

## 1. 目标

优先级规则用于决定玩家反馈的处理速度、通知范围和协同方式。MVP 阶段采用规则优先、AI 辅助判断的方式：先根据分类、关键词和影响面给出默认优先级，再结合情绪、玩家分层、同类反馈数量和版本集中度做升级或降级。

## 2. 优先级定义

| 优先级 | 定义 | 响应目标 | 飞书动作 |
| --- | --- | --- | --- |
| P0 | 影响支付、登录、崩溃、合规或舆情的严重问题 | 立即响应 | 群聊 @所有人 |
| P1 | 影响核心体验、多用户反馈或高价值玩家强负面问题 | 高优处理 | @负责人或值班同学 |
| P2 | 单用户可绕过问题、一般功能建议或中低频体验问题 | 常规跟进 | 写入反馈表和日报 |
| P3 | 普通规则咨询、低频偏好建议、观察类问题 | 记录观察 | 仅入库 |

## 3. 默认优先级

| 一级分类 | 二级分类 | 默认优先级 | 默认负责人 |
| --- | --- | --- | --- |
| 支付充值 | 充值未到账、重复扣费、退款争议、未成年人支付 | P0 | 支付/客服 |
| 账号安全 | 登录失败、账号找回、账号异常、实名问题 | P0/P1 | 账号/客服 |
| 处罚申诉 | 封禁、禁言、举报处罚争议 | P0/P1 | 风控/客服 |
| 活动奖励 | 奖励未到账、活动规则不清、任务未完成 | P1/P2 | 活动运营 |
| 游戏 BUG | 闪退、卡死、显示异常、结算异常 | P1/P2 | 客户端/测试 |
| 玩法规则 | 牌型、倍率、结算、入场门槛 | P3 | 玩法策划 |
| 举报投诉 | 外挂、辱骂、消极游戏、作弊怀疑 | P1/P2 | 风控/客服 |
| 体验建议 | 匹配、界面、引导、活动体验、玩法建议 | P2/P3 | 策划/运营 |

## 4. P0 触发条件

满足任一条件即判为 P0：

- 大量充值不到账、重复扣费。
- 未成年人支付、退款投诉、强合规风险。
- 大量无法登录、无法进入房间。
- 新版本大面积闪退、卡死。
- 明显舆情风险，例如“我要投诉”“发网上”“12315”“曝光”。
- 单位时间内同类反馈异常暴涨。
- 支付、账号、登录、崩溃类问题同时伴随明显负面情绪。

关键词示例：

```text
没到账、扣了两次、重复扣费、退款、未成年、12315、投诉、发网上、曝光、登录不了、进不去、闪退、卡死、崩了
```

## 5. P1 触发条件

满足任一条件可判为 P1：

- 活动奖励异常。
- 高频玩法争议。
- 多人反馈 BUG。
- 高价值玩家强负面反馈。
- 排行榜、公会、结算等影响核心体验的问题。
- 举报外挂、辱骂、消极游戏等影响公平性和社区氛围的问题。
- 单个问题尚未达到 P0，但出现明显扩散趋势。

关键词示例：

```text
奖励没发、任务完成没算、结算错了、排行榜异常、有人开挂、骂人、牌局异常、公会奖励、连续几局
```

## 6. P2 触发条件

满足以下条件通常判为 P2：

- 单用户可绕过问题。
- 一般功能建议。
- 中低频体验问题。
- 单人任务卡住、道具显示异常。
- 活动规则理解偏差，但未产生大面积负面情绪。

## 7. P3 触发条件

满足以下条件通常判为 P3：

- 普通规则咨询。
- 个性化偏好建议。
- 低频非阻断建议。
- 已有知识库可直接回答的问题。
- 没有明显负面情绪、风险和处理诉求的咨询。

## 8. 升级规则

| 条件 | 升级动作 |
| --- | --- |
| 30 分钟内同类反馈数达到 5 条 | P2 升 P1 |
| 30 分钟内同类反馈数达到 10 条 | P1 升 P0 |
| 同类反馈集中在同一新版本 | 优先级上调一级 |
| 高价值/付费玩家明显负面 | 优先级上调一级 |
| 出现投诉、曝光、监管关键词 | 直接 P0 |
| 涉及未成年人支付 | 直接 P0 |

## 9. 降级规则

仅在信息充分时才可降级：

- 玩家问题已被知识库明确回答，且无后续处理诉求：降为 P3。
- 单用户体验建议，无情绪风险且影响面低：降为 P3。
- BUG 类反馈缺少版本、设备、场景且玩家未继续补充：保留 P2，不再触发告警。

## 10. 告警去重规则

同一项目 + 同一分类 + 同一子类 + 同一版本，在 30 分钟内只推送一条聚合告警。

聚合字段：

- duplicate_group_id。
- related_feedback_count。
- 首次触发时间。
- 最近更新时间。
- 当前优先级。
- 当前处理状态。

如果后续反馈数量继续快速上涨，自动升级优先级并发送升级告警。

## 11. 规则引擎伪代码

```text
输入：feedback

priority = category_default_priority(feedback.category, feedback.sub_category)

if has_p0_keyword(feedback.raw_question) or has_p0_risk(feedback.risk_flags):
  priority = P0

if feedback.category in ["支付充值", "账号安全"] and feedback.emotion == "明显负面":
  priority = max(priority, P0)

if feedback.category in ["活动奖励", "游戏 BUG", "举报投诉"] and is_multi_user_issue(feedback):
  priority = max(priority, P1)

if feedback.player_segment in ["高价值玩家", "付费玩家"] and feedback.emotion == "明显负面":
  priority = upgrade_one_level(priority)

if related_feedback_count >= 10 within 30 minutes:
  priority = P0
else if related_feedback_count >= 5 within 30 minutes:
  priority = max(priority, P1)

return priority
```

## 12. 人工复核边界

以下问题必须进入人工复核：

- 退款、赔付、补发。
- 账号找回、实名、账号安全。
- 封禁、处罚、申诉。
- 未成年人支付。
- 涉及外挂、欺诈、辱骂等举报。
- 任何带有监管投诉或舆情风险的内容。
