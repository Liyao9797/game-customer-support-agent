更新时间：2026-05-28

## 1. 页面入口

| 页面 | URL | 用途 |

| 玩家端 WebView | `http://127.0.0.1:5188/app/player.html` | 玩家咨询、提交反馈、查询进度 |

## 2. 手动验收链路

### 2.1 玩家端提交反馈

打开：

```text
http://127.0.0.1:5188/app/player.html
```

输入：

```text
新手赛事奖励没到账
```

预期：

- 玩家端只展示追问和反馈确认卡。
- 不展示内部优先级、负责人、工单列表、状态流转、飞书状态。

操作：

```text
点击“确认提交”
```

预期：

- 生成 `FB20260528xxxx` 格式反馈编号。

### 2.2 运营端查看工单

打开：

```text
http://127.0.0.1:5188/app/ops.html
```

预期：

- 左侧反馈列表中能看到刚才提交的反馈。
- 可以按优先级、状态、分类筛选。
- 点击反馈后，右侧显示玩家反馈原话、关键字段和时间线。

### 2.3 运营端流转状态

在运营端点击：

```text
标记已受理
开始处理
```

预期：

- 工单状态变为 `处理中`。
- 时间线新增状态变更记录。

### 2.4 玩家端查询进度

回到玩家端，输入：

```text
FB20260528xxxx 处理到哪了
```

预期：

- 玩家端展示当前状态为“处理中”。
- 不展示运营端内部时间线和负责人。
