# 玩家端流程验收用例

更新时间：2026-06-09  
来源文档：[PLAYER_FLOW_TEST_TREE.md](PLAYER_FLOW_TEST_TREE.md)
适用页面：`/app/player.html`

## 1. 使用方式

本文件把流程树拆成可执行验收用例。每个用例都包含：

- 初始状态：执行前玩家端应处于什么上下文。
- 用户动作：玩家点击或输入什么。
- 预期结果：Agent 应进入什么路由或回复什么。
- 禁止行为：出现这些内容即判定失败。
- 自动化方式：脚本如何检查。

默认验收顺序：

1. 快捷入口与推荐问题。
2. 玩法规则与活动规则。
3. 反馈收集和确认卡。
4. 兜底、转人工、截图和进度查询。

| PFT-004 | `idle` | 输入“三带倍数” | 追问“想了解哪个玩法的规则” | 直接回答某一玩法倍数 | VM 调用 `renderCardTypeIntentClarification` |
| PFT-005 | `idle` | 输入“暗牌三带倍数” | 回答“暗牌三带是 4 倍” | 回答“炸弹最大”这类泛压制说明 | 调用知识库检索和 `buildBriefAnswer` |

| PFT-007 | `knowledge_answered` | 点击“这局牌型有争议，怎么提交牌局核查？” | 进入反馈收集，第一步问玩家 UID | 继续解释牌型大小 | VM 调用 `handlePlayerMessage` |

| PFT-013 | `idle` | 输入“组队牌类牌型大小” | 命中组队牌类规则 | 串到三人牌局或暗牌规则 | 知识库检索结果检查 |

## 5. 反馈收集用例

| 用例 ID | 初始状态 | 用户动作 | 预期结果 | 禁止行为 | 自动化方式 |
| --- | --- | --- | --- | --- | --- |
| PFT-002 | `ready_to_confirm` | 关键信息收集完成 | 确认卡展示问题类型、摘要、已识别信息 | 展示“还缺少”；展示英文 key | VM 完成一条支付反馈并检查卡片 HTML |
| PFT-008 | `collecting_feedback` | 上传截图 | 截图计入反馈字段 | 截图丢失；要求重复上传 | VM 调用截图 input change |
| PFT-017 | `collecting_feedback` | 支付反馈逐步补充字段 | 每次只问一个字段 | 一次问 3 个以上字段 | VM 多轮对话检查 |
| PFT-018 | `collecting_feedback` | 举报投诉逐步补充字段 | 问昵称/UID 或座位方向、场次、时间、原因、证据 | 要求玩家提供不可见的牌局 ID | VM 多轮对话检查 |
| PFT-019 | `ready_to_confirm` | 点击“确认提交” | 生成反馈编号 | 提交后没有编号 | VM 检查 `FeedbackStore.addFeedback` |
