# 玩家端 WebView 交互规格

更新时间：2026-05-28  
状态：设计文档，暂不进入界面重构  
目标：定义玩家从游戏内客服入口进入后看到的轻量客服体验。

## 1. 页面定位

玩家端是一个游戏内 WebView，不是后台，不是工单系统，也不是调试台。它的目标是：

- 让玩家快速问规则。
- 让玩家按最少步骤提交问题。
- 让玩家拿到反馈编号。
- 让玩家能查处理进度。

## 2. 页面结构

```text
顶部栏
  - 标题：智能客服
  - 返回/关闭入口，由游戏 WebView 容器提供

快捷入口区
  - 玩法规则
  - 活动奖励
  - 充值支付
  - BUG/举报/建议

聊天区
  - Agent 欢迎语
  - 玩家消息
  - Agent 回答
  - 追问消息
  - 反馈确认卡
  - 提交成功卡

底部输入区
  - 文本输入
  - 发送按钮
  - 附件/截图入口，MVP 可先占位
```

## 3. 首页状态

欢迎语：

```text
你好，我是示例棋牌项目智能客服。你可以问我玩法规则、活动奖励、充值到账、BUG、举报或建议问题。涉及账号、订单、奖励和牌局争议时，我会帮你整理信息并提交给客服同学处理。
```

快捷入口：

| 入口 | 示例问题 |
| --- | --- |
| 玩法规则 | 黄金牌活动炸弹怎么算？ |
| 活动奖励 | 连续挑战活动鱼票会清空吗？ |
| 充值支付 | 我充值了 30 元没到账 |
| BUG/举报/建议 | 对局卡死了 / 有人开挂 / 匹配太慢了 |

## 4. 规则问答体验

玩家问：

```text
黄金牌活动飞机能压三带吗？
```

Agent 应回答：

- 先直接回答。
- 再补充必要边界。
- 不展示知识库 ID、置信度、内部分类。

推荐结构：

```text
可以。黄金牌活动规则中，飞机可以压制三带。

如果你问的是某一局具体结算，请提供牌局 ID 和截图，我可以帮你提交给客服核查。
```

## 5. 反馈追问体验

玩家问：

```text
我充值了 30 元没到账
```

Agent 应追问：

```text
为了帮你核查充值问题，请补充充值时间、支付渠道、购买的道具或礼包，以及订单号或支付截图。
```

玩家端不直接展示：

- P0。
- payment risk flag。
- owner。
- sync_status。

但 Agent 可以用更温和的语言表达：

```text
充值问题我会优先帮你提交核查。
```

## 6. 反馈确认卡

玩家端反馈确认卡应展示：

| 字段 | 是否展示 | 示例 |
| --- | --- | --- |
| 问题类型 | 展示 | 充值支付 |
| 问题摘要 | 展示 | 反馈充值后道具未到账 |
| 已补充信息 | 展示 | 金额 30 元、渠道 App Store |
| 还缺信息 | 展示 | 订单号或支付截图 |
| 提交说明 | 展示 | 提交后客服会继续核查 |
| 内部优先级 | 不展示 | P0/P1 不给玩家看 |
| 内部负责人 | 不展示 | 支付/客服不展示 |
| 字段完整度 | 不展示 | high/low 不展示 |

按钮：

- 确认提交。
- 继续补充。

## 7. 提交成功卡

提交成功后展示：

```text
已帮你提交反馈，反馈编号：FB202605280001。
客服同学会根据你提供的信息继续核查。你之后可以用这个编号查询处理进度。
```

可选操作：

- 复制反馈编号。
- 查询进度。
- 继续提问。

## 8. 进度查询

玩家输入：

```text
FB202605280001 处理到哪了
```

展示：

| 内部状态 | 玩家文案 |
| --- | --- |
| submitted | 已提交，正在排队处理 |
| triaged | 已受理，系统已完成问题分类 |
| processing | 相关同学正在处理中 |
| waiting_player | 还需要你补充一些信息 |
| resolved | 已有处理结论，请查看客服回复 |
| closed | 该反馈已关闭 |

不展示：

- 内部 timeline 明细。
- 内部负责人。
- 飞书同步状态。
- 告警去重信息。

## 9. 空状态与异常状态

| 场景 | 文案 |
| --- | --- |
| 未识别问题 | 我还没完全理解你的问题，可以选择下方入口，或补充玩法/活动/订单/截图信息。 |
| 未找到知识 | 当前知识库暂未找到确定规则，你可以补充截图或提交反馈。 |
| 查不到反馈编号 | 暂未找到这个反馈编号，请确认是否输入正确。 |
| 信息不足 | 我还需要补充几个信息，才能帮你提交给客服核查。 |

## 10. 玩家端验收标准

| 验收点 | 标准 |
| --- | --- |
| 快速入口 | 4 类常见入口清晰可见 |
| 规则回答 | 能直接回答，不暴露内部知识 ID |
| 反馈追问 | 一次追问不超过 5 个字段 |
| 确认卡 | 玩家能理解自己将提交什么 |
| 进度查询 | 玩家能用反馈编号查状态 |
| 权限边界 | 不展示内部优先级、负责人、工单列表和流转按钮 |
