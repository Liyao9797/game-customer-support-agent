## 3. 问题详情

### UX-20260528-001：玩家 Web 端缺少设计审美

| 字段 | 内容 |
| --- | --- |
| 建议优先级 | P1 |
| 当前状态 | 已处理，待评测 |
| 来源 | 用户反馈 |
| 发现日期 | 2026-05-28 |

具体问题：

当前玩家端虽然已经拆出 WebView，但视觉层面仍偏基础页面，缺少游戏客服入口应有的品牌感、空间节奏、组件精致度和移动端 WebView 质感。

优先级分析：

这个问题不阻断问答、反馈提交和进度查询，但会明显影响玩家端可信度，也会影响后续进入视觉设计阶段的输入质量。它属于“产品观感与使用信任”问题，建议标为 P1，后续进入界面优化阶段时优先处理。

处理结果：

已将玩家端重做为手机 WebView 智能客服界面：桌面打开时模拟手机容器，手机打开时全屏适配；优化了顶部客服栏、快捷入口、消息气泡、反馈卡和底部输入区。后续需要通过人工评测确认视觉观感、移动端可读性和交互顺畅度。

### UX-20260528-002：快捷入口交互预期不一致

| 字段 | 内容 |
| --- | --- |
| 建议优先级 | P1 |
| 当前状态 | 已处理，待评测 |
| 来源 | 用户反馈 |
| 发现日期 | 2026-05-28 |

具体问题：

玩家端点击“玩法规则 / 活动奖励 / 充值支付 / BUG 举报建议”等快捷入口后，当前行为是把一个示例问题填入输入框；用户预期是 Agent 继续追问“你在这个分类下具体遇到了什么问题”，而不是随机给出示例问题。

优先级分析：

这个问题会影响玩家端核心交互理解。快捷入口是玩家进入客服后的高频操作，如果行为不符合预期，会让玩家觉得系统在替自己输入问题，而不是引导表达诉求。它影响核心体验但不阻断完整链路，建议标为 P1。

处理结果：

已将玩家端快捷入口从“点击后填入示例问题”改为“点击分类后由 Agent 追问具体问题，并提供本分类常见问题按钮”。后续需要通过人工评测确认流程是否符合预期。

### UX-20260528-003：玩家输入需要增加模糊词理解

| 字段 | 内容 |
| --- | --- |
| 建议优先级 | P1 |
| 当前状态 | 待评估，暂不处理 |
| 来源 | 用户反馈 |
| 发现日期 | 2026-05-28 |

具体问题：

玩家 Web 端对玩家输入的自然语言还不够“游戏化”。玩家可能会把游戏内货币说成“金币”“豆”“欢乐豆”，这几类表达应统一理解为游戏内金币。玩家说“钱”时，需要结合上下文判断是游戏内金币，还是现实人民币/充值金额。

优先级分析：

这会影响意图识别、实体抽取和反馈分类准确性。尤其是“钱没到账”“钱少了”这类表达，既可能是充值人民币问题，也可能是游戏内金币/欢乐豆结算问题。如果不处理，容易把普通游戏币问题误判为支付充值 P0，或把支付问题误判为游戏内结算问题。它影响核心 Agent 判断质量，建议标为 P1。

暂不处理原因：

当前先记录语言理解规则，后续应在意图识别层增加同义词归一化和上下文消歧规则，再统一更新测试用例。

### UX-20260528-004：Agent 回复缺少人情味

| 字段 | 内容 |
| --- | --- |
| 建议优先级 | P2 |
| 当前状态 | 待评估，暂不处理 |
| 来源 | 用户反馈 |
| 发现日期 | 2026-05-28 |

具体问题：

当前 Agent 回复偏功能性，能回答和追问，但语气较机械。对于充值不到账、奖励未到账、卡死、举报等有明显负面情绪的场景，需要增加更有安抚感和服务感的话术，例如先表达理解，再说明会帮玩家整理信息并提交核查。

优先级分析：

该问题不影响链路可用性，但会影响客服体验和玩家信任感。尤其游戏客服场景里，玩家经常带着不满情绪进入客服入口，回复缺少人情味会削弱“智能客服”的真实感。建议标为 P2，排入话术优化。

暂不处理原因：

当前优先保证双端链路和数据结构稳定。后续可基于情绪识别结果，为不同场景增加更自然的回复模板。

## 4. 后续使用方式

- 每发现一个体验问题，新增一行。
- “建议优先级”只表达处理顺序，不等于立即开发。
- 真正进入开发前，需要在需求排期或当前迭代目标中明确选中。
- 后续如接入飞书多维表格，可从 `data/ux_issue_tracker.csv` 导入。
